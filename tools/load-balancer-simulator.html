◆ Run these four experiments

1

Overload it. Push traffic to ~200 with 2 servers — requests start dropping. Now switch algorithms: notice it barely helps. You're capacity-bound, not balance-bound. Flip on Auto-scaling and watch drops disappear.

2

Fail a server. With steady traffic, click a healthy server to kill it. The balancer stops routing there (health check) and spreads its load across the rest — latency spikes, then recovers.

3

Feel the difference. Compare Round Robin vs Least Connections under heavy load. Least-Conn steers around the busy servers; Round Robin keeps feeding them regardless.

4

The IP Hash trap. Switch to IP Hash — one "hot" client pins to a single server and overloads it while others idle. Kill a server and watch keys remap. This is exactly the problem consistent hashing fixes — your next tool.
