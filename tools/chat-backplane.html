Module 10 · WebSockets & scaling them

Scaling a chat server: the backplane

A websocket is a live connection held in one server's memory. That's fine on one server — but the moment you run two, they become islands that can't see each other's users. Click a person to send a message, and flip the backplane on and off to watch the wall appear and vanish.

02 What you're seeing

Backplane OFF — each server only knows about the sockets connected to itself. A message reaches people on the same server and simply vanishes for everyone on another server. Add a third server and it gets worse: three islands.

Backplane ON — every server publishes each message to a shared Redis channel that all servers subscribe to. The message goes up to Redis and back down to whichever server holds each recipient. Now anyone reaches anyone, and you can add as many servers as you like.

In the AksharShruti code

The "islands" were a plain dict — connected_rooms = {} — living in one process. The fix: the socket now await _redis.publish("chat:{conv}", …), and a redis_subscriber() on every server delivers incoming messages to its local sockets. Verified live: message from a user on :8001 reached a user on :8002. Wall gone.
