◆ Run these experiments

1

Turn retrieval off. Ask "Refund policy?" with RAG on, read the sourced answer, then flip Retrieval off and re-run. Same model, but now it guesses "14-day money-back" — a confident, wrong specific it invented. That gap is hallucination, and grounding is what closes it.

2

Crank k up. On "Free-tier storage?" set k = 1 (just the bull's-eye chunk), then slide to k = 6. The answer doesn't improve — but C8 "free trial" and other off-topic chunks get pulled in (amber "noise" tags) and the context-token count climbs. More k past the relevant chunk = more cost, more distraction.

3

Watch the near-miss. On "Free-tier storage?", note C8 scores ~21% because it shares the word "free" — but it's about the trial, not the tier. This is why real systems add re-ranking and hybrid (keyword + vector) search: raw similarity alone lets look-alikes sneak in.

4

Read the prompt, not just the answer. Expand the Assembled prompt panel: the retrieved chunks are literally pasted in as context with a "answer only from the context" system instruction. RAG isn't magic — it's disciplined prompt construction plus a good retriever.
