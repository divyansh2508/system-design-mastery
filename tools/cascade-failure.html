Cascading Failure & Circuit Breaker

A live service dependency graph — Frontend → API → Service B/C → Database. Inject a slowdown into any node and watch what happens with no protection: callers block on the failing dependency, thread pools exhaust, and the failure climbs upstream until the whole system is red. Then flip on Circuit Breaker + Bulkhead and watch the same fault get contained — the blast radius stops, and the rest of the system stays up on fallbacks.

◆ Run these four experiments

1

Kill the Database. Protection off, click Database. Watch the red climb upstream: Service B's threads block on the DB and its pool exhausts, then API's, then the Frontend — success rate craters toward 0. Notice Service C stays green but nothing reaches it: a healthy service, stranded.

2

Contain it. Leave the DB failing and flip Circuit Breaker + Bulkhead ON. The breaker trips open and calls fail fast; bulkheads keep each dependency in its own pool. The blast radius stops at Service B — API and Frontend recover to ~95% on fallback data.

3

The minor dependency trap. Protection off, inject Service C — a non-critical recommendations widget. A thread blocked on C can serve nothing else, so the shared pool fills and the whole site dies for a feature nobody needs. Now flip protection on: contained.

4

Faults, not load. With no fault, drag Traffic to max — everything stays green. This is a failure tool, not a capacity tool: the system is provisioned for the load. It's the broken dependency, not the traffic, that triggers the cascade.
